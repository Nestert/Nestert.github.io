# Детальный план: Сайт-портфолио

## A. Короткое описание идеи

Минималистичный сайт-портфолио художника с навигацией по категориям работ (проекты, живопись, графика и др.). Реализован как статический сайт с административной панелью для самостоятельного добавления/редактирования контента через GitHub. Включает галерею с лайтбоксом для просмотра изображений в увеличении, полностью адаптивен для мобильных устройств. Архитектура предусматривает простое расширение на мультиязычность в будущем.

---

## B. Цели и не-цели

**Цели:**
- Создать сайт-портфолио с категоризированным контентом
- Обеспечить самостоятельное управление контентом без программирования
- Реализовать галерею с лайтбоксом (просмотр в увеличении)
- Сделать полностью адаптивный дизайн
- Разместить на GitHub Pages
- Подготовить архитектуру для мультиязычности

**Не-цели:**
- Блог/новости (только портфолио)
- Корзина/покупки (нет e-commerce)
- Сложные анимации/3D
- Интеграция с соцсетями
- Система комментариев
- SEO-оптимизация (можно добавить позже)

## C. Пользовательские сценарии

1. **Посетитель сайта** заходит на главную страницу, видит навигацию и работы из первой категории
2. **Посетитель** кликает на "Живопись" в меню — видит все работы этой категории в виде сетки миниатюр
3. **Посетитель** кликает на миниатюру — открывается лайтбокс с увеличенным изображением, описанием и навигацией между работами
4. **Посетитель** на мобильном телефоне видит адаптированную версию сайта с удобным меню
5. **Администратор** заходит в /admin, авторизуется через GitHub, добавляет новую работу: загружает фото, пишет название, выбирает категорию, сохраняет
6. **Администратор** редактирует существующую работу или удаляет её
7. **Администратор** добавляет новую категорию в навигацию через конфигурацию
8. **Администратор** добавляет переводы на второй язык через админ-панель

## D. Требования

### Functional

- Навигация по категориям в верхней панели (проекты, живопись, графика + настраиваемые)
- Главная страница со списком категорий или последними работами
- Страница категории с сеткой миниатюр работ
- Лайтбокс: просмотр изображения в увеличении, стрелки навигации, закрытие по клику/Esc
- Админ-панель /admin для управления контентом (CRUD для работ)
- Загрузка изображений через админ-панель (интеграция с GitHub LFS для больших файлов)
- YAML/JSON файлы для хранения метаданных работ
- 404 страница
- "Наверх" кнопка на длинных страницах

### Non-functional

- **Performance**: Lighthouse 90+, ленивая загрузка изображений, WebP формат
- **Security**: HTTPS (GitHub Pages), CSRF-токены в админке, rate limiting на API
- **Privacy**: нет внешних трекеров/cookies, локальные шрифты
- **Offline**: PWA-кеширование базовых ресурсов
- **i18n**: архитектура для 2+ языков, UTF-8, RTL-поддержка не нужна
- **Accessibility**: WCAG AA, alt-тексты, клавиатурная навигация в галерее, focus states
- **Mobile**: 100% адаптивность, touch-свайп в лайтбоксе
- **Browser**: последние 2 версии Chrome, Safari, Firefox, Edge

## E. Предположения и риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| GitHub LFS лимиты для больших изображений | Средняя | Средняя | Оптимизация изображений перед загрузкой, Cloudinary как альтернатива |
| Сложность настройки Decap CMS для новичка | Низкая | Средняя | Подробная документация, скринкасты в README |
| GitHub Pages медленный в некоторых регионах | Низкая | Низкая | CDN включён, альтернатива — Netlify |
| Конфликты при одновременном редактировании | Очень низкая | Низкая | Git branching, уведомления в CMS |
| Лайтбокс не работает на старых iOS | Низкая | Низкая | Graceful degradation, тестирование |

### Модели данных

**Work (работа):**
```yaml
---
id: work-001
title: "Название работы"
category: paintings
year: 2024
description: "Описание работы"
description_en: "English description"
image: /assets/images/paintings/work-001.jpg
thumbnail: /assets/images/paintings/work-001-thumb.jpg
order: 1
---
```

**Category (категория):**
```yaml
---
slug: paintings
title: "Живопись"
title_en: "Paintings"
icon: brush
order: 2
---
```

### API/Эндпоинты

- `/` — главная страница
- `/[category]/` — страница категории
- `/[category]/[work-slug]/` — отдельная работа (опционально)
- `/admin/` — админ-панель Decap CMS
- `/sw.js` — Service Worker

## G. План работ по этапам

### Этап 1: Базовая структура проекта
**Задачи:**
1. Создать репозиторий на GitHub
2. Настроить npm-проект, установить 11ty, Sass
3. Создать базовые директории и конфиги
4. Настроить GitHub Actions для деплоя на Pages
5. Инициализировать git, первый коммит

**DoD:**
- `npm run dev` запускает локальный сервер
- `npm run build` генерирует статический сайт в `_site/`
- `npm run deploy` деплоит на GitHub Pages
- Сайт открывается по URL `username.github.io/repo/`

### Этап 2: Базовая вёрстка и навигация
**Задачи:**
1. Создать базовый HTML-шаблон (Nunjucks)
2. Написать CSS (переменные, типографика, сетка)
3. Сверстать хедер с навигацией
4. Сверстать футер
5. Реализовать мобильное меню (гамбургер)
6. Добавить базовую 404 страницу

**DoD:**
- Сайт визуально похож на референс (минимализм, шрифты)
- Меню работает на десктопе и мобильном
- Адаптивность проверена на 320px–1920px
- Lighthouse Accessibility ≥ 90

### Этап 3: Система контента и категорий
**Задачи:**
1. Создать структуру `content/` с YAML-файлами
2. Написать 11ty-коллекции для категорий
3. Создать шаблоны страниц категорий
4. Добавить тестовые данные (3-5 работ на категорию)
5. Реализовать вывод галереи миниатюр

**DoD:**
- Категории отображаются в меню
- При клике открывается страница с работами
- Работы отсортированы по полю `order`
- Пустые категории не показываются

### Этап 4: Галерея и лайтбокс
**Задачи:**
1. Написать CSS Grid для сетки миниатюр
2. Реализовать lazy loading изображений
3. Создать lightbox-компонент (HTML/CSS/JS)
4. Добавить навигацию стрелками в лайтбоксе
5. Добавить свайп на мобильных
6. Реализовать закрытие по Esc и клику вне

**DoD:**
- Клик на миниатюру → открывается лайтбокс
- Работает клавиатурная навигация
- Работает touch-свайп на мобильном
- Анимации плавные (60fps)

### Этап 5: Админ-панель (Decap CMS)
**Задачи:**
1. Создать `/admin/index.html` и `config.yml`
2. Настроить бэкенд GitHub Gateway
3. Создать коллекции для категорий и работ
4. Настроить поля: заголовок, категория, изображение, описание
5. Добавить медиа-библиотеку
6. Протестировать создание/редактирование/удаление

**DoD:**
- /admin открывается, предлагает авторизацию через GitHub
- Можно создать новую работу с загрузкой изображения
- Изображение коммитится в репозиторий
- Изменения появляются на сайте после деплоя

### Этап 6: Оптимизация и полировка
**Задачи:**
1. Добавить image pipeline (оптимизация, WebP, srcset)
2. Настроить Service Worker для offline
3. Добавить meta tags (title, description, og:image)
4. Проверить Accessibility (Axe, Lighthouse)
5. Написать README с инструкциями
6. Добавить документацию по админке

**DoD:**
- Lighthouse Performance ≥ 90
- Lighthouse Accessibility = 100
- Сайт работает без интернета (кеш)
- README понятно описывает использование

## H. Тест-план

### Unit-тесты
- Валидация YAML-файлов (js-yaml)
- Генерация URL-ов из slug-ов
- Сортировка работ по полю `order`

### Integration-тесты
- Рендер всех страниц без ошибок
- Все ссылки ведут на существующие страницы
- Корректная работа навигации по категориям
- Соответствие метаданных в HTML и YAML

### E2E-тесты (Playwright)
- Главная страница загружается
- Переход по категориям работает
- Открытие/закрытие лайтбокса
- Навигация стрелками в лайтбоксе
- Mobile view: меню открывается, галерея адаптируется

### Ручное тестирование
- Safari (macOS + iOS)
- Chrome (Android + десктоп)
- Firefox
- Проверка загрузки больших изображений
- Проверка workflow: добавить работу → коммит → деплой → проверить на сайте

## I. Правила реализации

### Стиль и конвенции

**CSS:**
- Методология: BEM-подобные классы (блок__элемент--модификатор)
- Препроцессор: SCSS
- Единицы: `rem` для типографики, `vw/vh` для layout при необходимости
- Переменные в `:root` для цветов, отступов, шрифтов
- Mobile-first подход

**JavaScript:**
- Vanilla JS (ES6+), без jQuery
- Деградация: если JS отключён — сайт работает
- ESLint + Prettier

**Git:**
- Ветки: `main` (production), `develop` (development), `feature/*` (фичи)
- Коммиты: Conventional Commits (`feat:`, `fix:`, `docs:`, `refactor:`)
- PR: минимум 1 approve, тесты зелёные, линтер проходит
